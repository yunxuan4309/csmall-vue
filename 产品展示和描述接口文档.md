# 前端接口文档 —— 产品展示 + 秒杀模块

## 通用说明

### 统一响应格式

所有接口返回统一的 JSON 结构：

```json
{
  "state": 200,       // 业务状态码：200成功，400参数错误，401未登录，403无权限，404不存在，500服务器错误
  "message": "操作成功",
  "data": {}          // 业务数据
}
```

### 分页响应格式

分页接口的 `data` 结构：

```json
{
  "page": 1,           // 当前页码
  "pageSize": 10,      // 每页条数
  "totalPage": 5,      // 总页数
  "total": 48,         // 总记录数
  "list": []           // 当前页数据列表
}
```

### 认证方式

除特别标注外，所有接口均需在请求头携带 Token：

```
Authorization: Bearer {token}
```

Token 通过 SSO 登录接口获取：
- 用户登录：`POST http://localhost:10002/sso/user/login`
- 管理员登录：`POST http://localhost:10002/sso/admin/login`

| 403 | 您已经购买过这个商品了 | 重复购买 |
| 404 | 随机码不正确 | randCode 错误或过期 |
| 404 | 没有找到当前商品的随机码 | 秒杀未开始或已结束 |
| 400 | 对不起，您要购买的商品已经售罄 | 库存不足 |
| 500 | 没有该商品缓存信息 | 缓存真空期，稍后重试 |

## 四、前端页面开发建议流程

```
1. 登录页 → 调用 3.1 获取 Token
2. 首页/商品列表 → 调用 1.1 获取分类 + 1.2 获取商品列表
3. 商品详情页 → 调用 1.3 商品信息 + 1.4 详情图片 + 1.5 SKU列表
4. 秒杀列表页 → 调用 2.1 秒杀商品列表（对比 listPrice 和 seckillListPrice 展示折扣）
5. 秒杀详情页 → 调用 2.2 秒杀商品 + 2.3 详情图片 + 2.4 秒杀SKU
6. 秒杀下单 → 判断 url 字段非空 → 调用 2.5 提交订单
```
