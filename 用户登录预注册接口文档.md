# 前端接口文档 —— 普通用户注册与登录

## 通用说明

### 统一响应格式

所有接口返回统一的 JSON 结构：

```json
{
  "state": 200,
  "message": "操作成功",
  "data": {}
}
```

| state | 含义 |
|-------|------|
| 200 | 成功 |
| 400 | 参数错误 / 业务失败 |
| 401 | 未登录 |
| 403 | 无权限 |
| 409 | 冲突（如重复注册、重复登出） |

### 认证方式

需认证的接口在请求头携带：

```
Authorization: Bearer {tokenValue}
```

## 一、注册相关接口（mall-ums，端口 10006）

### 1.1 校验用户名/手机号/邮箱是否已存在

> 建议注册前先调用此接口做实时校验，避免提交后才发现重复。

**请求**

```
POST http://localhost:10006/ums/user/checkValue
```

**权限**：无需登录

**参数**

| 参数 | 位置 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| value | Query/Form | String | 是 | 待校验的值 |
| type | Query/Form | String | 是 | 校验类型：`username` / `phone` / `email` |

**请求示例**

```
POST http://localhost:10006/ums/user/checkValue?value=testuser1&type=username
POST http://localhost:10006/ums/user/checkValue?value=13800138001&type=phone
POST http://localhost:10006/ums/user/checkValue?value=test1@example.com&type=email
```

**成功响应**（值可用，未重复）

```json
{
  "state": 200,
  "message": "操作成功"
}
```

**失败响应**（值已存在）

```json
{
  "state": 400,
  "message": "用户名已被使用"
}
```

```json
{
  "state": 400,
  "message": "手机号已被使用"
}
```

```json
{
  "state": 400,
  "message": "邮箱已被使用"
}
```

### 1.2 注册用户

**请求**

```
POST http://localhost:10006/ums/user/register
Content-Type: application/x-www-form-urlencoded
```

**权限**：无需登录

**参数**

| 参数 | 类型 | 必填 | 校验规则 | 错误提示 | 示例 |
|------|------|------|---------|---------|------|
| username | String | 是 | 4~16位，字母开头，仅含字母和数字 | 用户名必须是由字母、数字组成的4~16字符，且第1个字符必须是字母！ | `testuser1` |
| nickname | String | 是 | 2~16个字符 | 名称必须由2~16字符组成！ | `测试用户1` |
| phone | String | 是 | 11位手机号，1开头，第二位3/4/5/8/9 | 请填写正确的手机号！ | `13800138001` |
| email | String | 是 | 标准邮箱格式 | 请填写正确格式的邮箱！ | `test1@example.com` |
| password | String | 是 | 4~16位可打印ASCII字符 | 密码的长度必须是4~16位！ | `123456` |
| ackPassword | String | 是 | 与 password 一致 | 密码的长度必须是4~16位！ | `123456` |

**请求示例**

```
POST http://localhost:10006/ums/user/register
Content-Type: application/x-www-form-urlencoded

username=testuser1&nickname=测试用户1&phone=13800138001&email=test1%40example.com&password=123456&ackPassword=123456
```

**成功响应**

```json
{
  "state": 200,
  "message": "操作成功"
}
```

**失败响应示例**

```json
{
  "state": 400,
  "message": "注册用户失败，请填写用户名！"
}
```

```json
{
  "state": 400,
  "message": "注册用户失败，用户名必须是由字母、数字组成的4~16字符，且第1个字符必须是字母！"
}
```

```json
{
  "state": 400,
  "message": "注册用户失败，请填写正确的手机号！"
}
```

```json
{
  "state": 400,
  "message": "注册用户失败，请填写正确格式的邮箱！"
}
```

> 注册成功后不会自动登录，需要再调用登录接口获取 Token。

## 二、登录相关接口（mall-sso，端口 10002）

### 2.1 用户登录

**请求**

```
POST http://localhost:10002/user/sso/login
Content-Type: application/json
```

**权限**：无需登录

**参数**

| 参数 | 类型 | 必填 | 校验规则 | 示例 |
|------|------|------|---------|------|
| username | String | 是 | 4~16位，字母开头，仅含字母和数字 | `testuser1` |
| password | String | 是 | 4~16位可打印ASCII字符 | `123456` |

**请求示例**

```json
{
  "username": "testuser1",
  "password": "123456"
}
```

**成功响应**

```json
{
  "state": 200,
  "message": "操作成功",
  "data": {
    "tokenHeader": "Bearer",
    "tokenValue": "eyJhbGciOiJIUzI1NiJ9.eyJpZCI6MSwidXNlcm5hbWUiOiJ0ZXN0dXNlcjEiLCJ1c2VyVHlwZSI6IlVTRVIiLCJhdXRob3JpdGllcyI6WyJST0xFX3VzZXIiXX0.xxxxx"
  }
}
```

**TokenVO 字段说明**

| 字段 | 类型 | 说明 |
|------|------|------|
| tokenHeader | String | Token 头标识，固定值 `Bearer` |
| tokenValue | String | JWT Token 值 |

**前端使用方式**

```javascript
// 拼接 Token
const token = data.tokenHeader + ' ' + data.tokenValue
// 例如: "Bearer eyJhbGciOiJIUzI1NiJ9..."

// 后续请求携带
axios.defaults.headers.common['Authorization'] = token
```

**失败响应**

```json
{
  "state": 400,
  "message": "登录失败！用户名密码错误"
}
```

---

### 2.2 用户登出

**请求**

```
POST http://localhost:10002/user/sso/logout
Authorization: Bearer {token}
```

**权限**：需登录

**请求示例**

```
POST http://localhost:10002/user/sso/logout
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```

**成功响应**

```json
{
  "state": 200,
  "message": "操作成功"
}
```

**失败响应**

| state | message | 场景 |
|-------|---------|------|
| 404 | 当前客户端没有登录状态 | 未携带 Authorization 头 |
| 409 | 当前用户已经登出，不必重复登出 | 重复登出 |

## 三、修改密码（mall-ums，端口 10006）

### 3.1 修改登录密码

**请求**

```
POST http://localhost:10006/ums/user/renew/password
Authorization: Bearer {token}
Content-Type: application/x-www-form-urlencoded
```

**权限**：需登录（普通用户 `ROLE_user`）

**参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 详见 ChangePasswordDTO | - | - | 包含旧密码、新密码、确认密码等字段 |

> 此接口需先登录获取 Token 后才能调用。

## 四、完整流程图

```
┌──────────────────────────────────────────────────┐
│                    注册流程                        │
│                                                    │
│  1. 输入用户名 → 调用 1.1 checkValue(type=username) │
│  2. 输入手机号 → 调用 1.1 checkValue(type=phone)    │
│  3. 输入邮箱   → 调用 1.1 checkValue(type=email)    │
│  4. 填写密码   → 前端校验格式+确认密码一致性          │
│  5. 提交注册   → 调用 1.2 register                  │
│  6. 注册成功   → 跳转登录页                         │
└──────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────┐
│                    登录流程                        │
│                                                    │
│  1. 输入用户名+密码 → 调用 2.1 login               │
│  2. 获取 tokenHeader + tokenValue                  │
│  3. 拼接: "Bearer " + tokenValue                   │
│  4. 存储到 localStorage / Vuex / Pinia             │
│  5. 后续请求 Header: Authorization: Bearer xxx     │
│  6. 退出登录 → 调用 2.2 logout，清除本地 Token      │
└──────────────────────────────────────────────────┘
```

## 六、JWT Token 说明

Token 中包含的用户信息（解码 JWT payload 可得）：

| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| id | Long | 用户ID | 1 |
| username | String | 用户名 | `testuser1` |
| userType | String | 用户类型 | `"USER"` |
| authorities | List\<String\> | 权限列表 | `["ROLE_user"]` |

**Token 有效期**：7天（604800秒），配置于各模块 `application.yml` 中 `jwt.expiration`。
