## 第一层：环境配置（.env）

你的后端是微服务架构，不同模块可能走不同端口。开发环境建议统一走网关，生产环境走 Nginx 反向代理。

```
mall-vue/
├── .env                          # 所有环境共享的默认值
├── .env.development              # 开发环境
├── .env.staging                  # 测试环境
└── .env.production               # 生产环境
```

**`.env`** — 公共默认值：

```
# Token 在 localStorage 中的 key
VITE_TOKEN_KEY=mall_token

# Token 前缀
VITE_TOKEN_PREFIX=Bearer

# Token 有效期（秒），用于前端主动续期判断
VITE_TOKEN_EXPIRATION=604800
```

**`.env.development`** — 开发环境：

```
# 开发环境：各服务直连（因为 SSO 未注册网关，建议开发期直连）
VITE_API_SSO=http://localhost:10002
VITE_API_GATEWAY=http://localhost:10087
VITE_API_ADMIN=http://localhost:10002
```

**`.env.production`** — 生产环境：

```
# 生产环境：统一走 Nginx 反向代理
VITE_API_BASE_URL=/api
```

在代码中通过 `import.meta.env.VITE_API_SSO` 读取，Vite 会自动注入。

## 第二层：Axios 封装（lib/http.js）

针对你的后端返回结构 `JsonResult<T>`（`{ state, message, data }`）做统一封装：

```javascript
import axios from 'axios'

// ============ 创建实例工厂 ============

/**
 * 创建 Axios 实例
 * @param {string} baseURL - 服务基础地址
 */
function createInstance(baseURL) {
  const instance = axios.create({
    baseURL,
    timeout: 15000,
    headers: { 'Content-Type': 'application/json' }
  })

  // ---- 请求拦截 ----
  instance.interceptors.request.use(
    config => {
      const token = localStorage.getItem(import.meta.env.VITE_TOKEN_KEY)
      if (token) {
        config.headers.Authorization =
          `${import.meta.env.VITE_TOKEN_PREFIX} ${token}`
      }
      return config
    },
    error => Promise.reject(error)
  )

  // ---- 响应拦截 ----
  instance.interceptors.response.use(
    response => {
      const res = response.data

      // 判断后端 JsonResult.state
      if (res.state === 200) {
        // 成功：只返回 data 部分，页面拿到的就是纯业务数据
        return res
      }

      // Token 过期或未认证
      if (res.state === 401) {
        localStorage.removeItem(import.meta.env.VITE_TOKEN_KEY)
        // 避免登录页死循环
        if (window.location.hash !== '#/admin/login'
            && window.location.hash !== '#/login') {
          window.location.hash = '#/admin/login'
        }
        return Promise.reject(new Error(res.message || '未登录'))
      }

      // 其他业务错误（400/403/404/409/500）
      return Promise.reject(new Error(res.message || '请求失败'))
    },
    error => {
      // HTTP 层面错误（网络断开、超时、跨域等）
      const msg = error.response
        ? `服务器错误 ${error.response.status}`
        : error.message || '网络异常'
      return Promise.reject(new Error(msg))
    }
  )

  return instance
}

// ============ 导出各服务实例 ============

// SSO 服务（登录相关，端口 10002）
export const ssoHttp = createInstance(import.meta.env.VITE_API_SSO)

// 网关（其他业务请求统一走网关）
export const gatewayHttp = createInstance(import.meta.env.VITE_API_GATEWAY)

// 默认导出（给不需要区分服务的场景）
export default gatewayHttp
```

## 第三层：API 模块层

## 第四层：页面调用

页面层通过 `await` 调用 API，因为拦截器已经处理了 `JsonResult`，拿到的结构是：

```javascript
// 成功时拦截器返回完整 JsonResult：{ state: 200, data: {...} }
// 页面只需取 .data 即可

import { adminLogin, getUserInfo } from '@/api/sso'

async function handleLogin() {
  try {
    const res = await adminLogin(form.username, form.password)
    // res = { state: 200, message: null, data: { tokenHeader, tokenValue } }

    // 存储 Token
    localStorage.setItem(
      import.meta.env.VITE_TOKEN_KEY,
      res.data.tokenValue
    )

    // 获取用户信息
    const infoRes = await getUserInfo()
    // infoRes.data = { userId, username, nickname, phone }

    router.push('/admin/dashboard')
  } catch (err) {
    ElMessage.error(err.message)  // 直接拿拦截器抛出的错误信息
  }
}
```

## 第五层：可选的统一类型约束（TypeScript）

如果你后续迁移到 TypeScript，可以定义泛型类型来约束返回体：

```typescript
// types/api.d.ts

/** 后端统一返回结构，对应 JsonResult<T> */
interface JsonResult<T = any> {
  state: number
  message: string | null
  data: T
}

/** Token 返回结构 */
interface TokenVO {
  tokenHeader: string
  tokenValue: string
}

/** 用户信息结构 */
interface UserInfoVO {
  userId: number
  username: string
  nickname: string
  phone: string
}

/** 分页结构，对应 JsonPage<T> */
interface JsonPage<T = any> {
  page: number
  pageSize: number
  totalPage: number
  total: number
  list: T[]
}
```

## 总结对照表

| 层级 | 文件/目录 | 职责 | 对接后端 |
|---|---|---|---|
| 环境配置 | `.env.*` | 不同环境的 API 地址 | - |
| 实例+拦截 | `lib/http.js` | Token 注入、JsonResult 解包、401 跳转 | `JsonResult` 结构 |
| API 模块 | `api/*.js` | 按业务模块声明接口 | 各 Controller 路径 |
| 页面调用 | `views/*.vue` | 调用 API、处理 UI | - |

核心原则是：**拦截器吃掉 `JsonResult` 的外壳，API 模块只映射路径，页面只拿 `data` 做业务**。
