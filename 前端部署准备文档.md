## 一、后端服务地址

### 生产环境

| 项目 | 地址 |
|------|------|
| API 基地址 | `http://8.156.85.160` |
| 备案后域名 | `http://coolshark-shop.cn`（待 ICP 备案后生效） |

## 二、接口路由规则

后端 Gateway 按路径前缀路由到不同服务：

| 路径前缀 | 目标服务 | 功能 |
|----------|----------|------|
| `/front/**` | mall-front | 前台商品展示、分类浏览 |
| `/sso/**` | mall-sso | 用户/管理员登录认证 |
| `/admin/sso/**` | mall-sso | 管理员登录 |
| `/user/sso/**` | mall-sso | 用户登录 |
| `/oms/**` | mall-order | 订单、购物车 |
| `/search/**` | mall-search | 商品搜索 |
| `/resource/**` | mall-resource | 文件上传、图片访问 |
| `/seckill/**` | mall-seckill | 秒杀活动 |

**不对外暴露的服务**（仅内部 Dubbo RPC 调用）：mall-ams、mall-ums、mall-product

### 请求示例

```
GET  http://8.156.85.160/front/category/list        → 商品分类列表
POST http://8.156.85.160/sso/admin/login             → 管理员登录
POST http://8.156.85.160/sso/user/login              → 用户登录
GET  http://8.156.85.160/search/search?keyword=手机   → 搜索商品
POST http://8.156.85.160/oms/order/create             → 创建订单
```

## 三、认证机制

### JWT Token

- 登录成功后，后端在响应头 `Authorization` 中返回 JWT Token
- 后续请求需在请求头中携带：`Authorization: Bearer <token>`
- Token 有效期由后端控制

### CORS 跨域

**重要**：后端 Gateway 已统一处理 CORS，前端无需单独配置跨域。

当前 Gateway 允许的来源：
- `http://localhost:5173`（本地开发）
- `http://127.0.0.1:5173`（本地开发）

**生产环境部署前端时**，需要后端在 Gateway 的 `CorsConfig.java` 中添加生产域名/IP 到 `allowedOrigin` 列表。前端部署完成后通知后端添加。

## 五、前端部署方案

### 方案 A：Nginx 同服务器部署（推荐）

将前端构建产物（dist 目录）放到服务器上，由同一个 Nginx 提供服务：

```
浏览器 → Nginx(80) → 前端静态文件 或 Gateway(10087)
```

Nginx 配置修改为：

```nginx
server {
    listen 80;
    server_name _;
    client_max_body_size 50m;

    # 前端静态文件
    location / {
        root /data/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;  # SPA 路由回退
    }

    # API 请求代理到 Gateway
    location /front/ {
        proxy_pass http://127.0.0.1:10087;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    location /sso/ {
        proxy_pass http://127.0.0.1:10087;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    location /oms/ {
        proxy_pass http://127.0.0.1:10087;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    location /search/ {
        proxy_pass http://127.0.0.1:10087;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    location /resource/ {
        proxy_pass http://127.0.0.1:10087;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    location /seckill/ {
        proxy_pass http://127.0.0.1:10087;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    location /admin/ {
        proxy_pass http://127.0.0.1:10087;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    location /user/ {
        proxy_pass http://127.0.0.1:10087;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 方案 B：前端部署到其他平台

如果前端部署到 Vercel / Netlify 等平台，需注意：
1. 配置 API 代理或直接用完整后端地址
2. 后端 Gateway 需要添加前端域名到 CORS 白名单
