## 秒杀模块接口文档

---

### 一、秒杀商品浏览（用户端）

#### 1. 获取秒杀商品列表

```
GET /seckill/spu/list?page=1&pageSize=10
```

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | Integer | 是 | 页码，从1开始 |
| pageSize | Integer | 是 | 每页数量 |

**请求头：** 需要携带 Authorization (Bearer Token)

**响应示例：**
```json
{
  "state": 200,
  "message": "ok",
  "data": {
    "records": [
      {
        "id": 1,
        "name": "iPhone 15 Pro",
        "title": "...",
        "listPrice": 7999.00,
        "seckillListPrice": 6999.00,
        "startTime": "2026-05-09 10:00:00",
        "endTime": "2026-05-09 12:00:00",
        "url": null
      }
    ],
    "total": 6,
    "current": 1,
    "size": 10
  }
}
```

> `url` 字段：仅在当前时间处于秒杀时间窗口内时返回值（如 `"/seckill/385721"`），窗口外为 `null`。前端根据 `url` 是否有值来显示"立即抢购"按钮。

---

#### 2. 获取秒杀商品详情

```
GET /seckill/spu/{spuId}
```

**路径参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| spuId | Long | 商品SPU id（pms_spu表的主键id） |

**响应示例：**
```json
{
  "state": 200,
  "message": "ok",
  "data": {
    "id": 1,
    "name": "iPhone 15 Pro",
    "title": "...",
    "listPrice": 7999.00,
    "seckillListPrice": 6999.00,
    "startTime": "2026-05-09 10:00:00",
    "endTime": "2026-05-09 12:00:00",
    "url": "/seckill/385721"
  }
}
```

---

#### 3. 获取秒杀商品详情页内容

```
GET /seckill/spu/{spuId}/detail
```

**路径参数：** 同上

**响应示例：**
```json
{
  "state": 200,
  "message": "ok",
  "data": {
    "detail": "<p>商品详情HTML内容...</p>"
  }
}
```

---

```
GET /seckill/sku/list/{spuId}
```

**路径参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| spuId | Long | 商品SPU id |

**响应示例：**

      "title": "iPhone 15 Pro 128GB 深空黑色",
      "price": 7999.00,

### 二、秒杀下单（用户端）

#### 5. 提交秒杀订单

```
POST /seckill/{randCode}
```

**路径参数：**

| 参数 | 类型 | 说明 |
|------|------|------|
| randCode | String | 随机码，从秒杀商品详情的 `url` 字段中获取 |

**请求头：** 需要携带 Authorization (Bearer Token)，角色必须为 user

**请求体：**
```json
{
  "spuId": 1,
  "seckillOrderItemAddDTO": {
    "skuId": 1,
    "title": "iPhone 15 Pro 128GB 深空黑色",
    "mainPicture": "http://...",
    "price": 6999.00,
    "quantity": 1,
    "barCode": "APL-IP15P-001",
    "data": "{}"
  }
}
```

**请求参数说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| spuId | Long | 是 | 商品SPU id |
| seckillOrderItemAddDTO.skuId | Long | 是 | 商品SKU id |
| seckillOrderItemAddDTO.title | String | 是 | SKU标题 |
| seckillOrderItemAddDTO.mainPicture | String | 否 | 商品主图URL |
| seckillOrderItemAddDTO.price | BigDecimal | 是 | 秒杀价格 |
| seckillOrderItemAddDTO.quantity | Integer | 是 | 购买数量（不超过seckillLimit） |
| seckillOrderItemAddDTO.barCode | String | 否 | 条形码 |
| seckillOrderItemAddDTO.data | String | 否 | 附加数据 |

**成功响应：**
```json
{
  "state": 200,
  "message": "ok",
  "data": {
    "id": 1001,
    "sn": "20260509100000001",
    "createTime": "2026-05-09 10:30:00",
    "payAmount": 6999.00
  }
}
```

**可能返回的错误码：**

| 状态 | message | 说明 |
|------|---------|------|
| 404 | 随机码不正确 | randCode与Redis中不匹配 |
| 404 | 没有找到当前商品的随机码 | 不在秒杀时间窗口或预热未完成 |
| 403 | 您已经购买过这个商品了 | 同一用户同一SKU限购1次 |
| 400 | 对不起，您要购买的商品已经售罄 | 库存不足 |
| 500 | 服务器忙,请稍后再试 | 触发Sentinel限流 |

> **前端调用要点**：
> 1. 先通过商品详情接口获取 `url`（如 `/seckill/385721`）
> 2. 从 `url` 中提取随机码（`385721`）
> 3. 将随机码作为路径参数，组装最终请求地址 `POST /seckill/385721`
> 4. `url` 为 `null` 时，隐藏或禁用抢购按钮

### 三、秒杀订单支付

#### 6. 支付秒杀订单

秒杀订单与普通订单存储在同一张 `oms_order` 表中，**支付接口与普通订单共用**：

```
POST /oms/order/pay
```

```json
{
  "id": 1001,
  "paymentType": 0
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | Long | 是 | 秒杀下单返回的订单id |
| paymentType | Integer | 否 | 支付方式：0=银联，1=微信，2=支付宝。默认0（模拟支付） |

**响应：**
```json
{
  "state": 200,
  "message": "ok",
  "data": {
    "id": 1001,
    "sn": "20260509100000001",
    "paymentType": 0,
    "payAmount": 6999.00,
    "gmtPay": "2026-05-09 10:31:00",
    "state": 3
  }
}
```

> `state=3` 表示已支付。

### 四、秒杀商品管理（管理员端）

#### 7. 新增秒杀SPU

```
POST /seckill/manage/spu
```

**权限：** admin

```json
{
  "spuId": 1,
  "listPrice": 6999.00,
  "startTime": "2026-05-09 10:00:00",
  "endTime": "2026-05-09 12:00:00"
}
```

#### 8. 删除秒杀SPU

```
DELETE /seckill/manage/spu/{id}
```

**路径参数：** `id` 为 seckill_spu 表的主键id

#### 9. 新增秒杀SKU

```
POST /seckill/manage/sku
```

**权限：** admin

```json
{
  "skuId": 1,
  "spuId": 1,
  "seckillStock": 50,
  "seckillPrice": 6999.00,
  "seckillLimit": 1
}
```

> `spuId` 填 seckill_spu 表的主键id（即新增SPU返回的id），不是 pms_spu 的id。

#### 10. 删除秒杀SKU

```
DELETE /seckill/manage/sku/{id}
```

**路径参数：** `id` 为 seckill_sku 表的主键id

### 五、前端对接注意事项

1. **随机码机制**：秒杀下单的URL是动态的，只有在秒杀时间窗口内才能从商品详情的 `url` 字段获取。窗口外该字段为 `null`，前端应隐藏抢购按钮
2. **限购**：同一用户同一SKU只能购买1次，重复购买会返回403错误
3. **售罄**：库存为0时返回400错误，前端可显示"已售罄"
4. **限流**：高并发场景下可能返回500"服务器忙"，前端应提示用户稍后重试
5. **时间判断**：前端可根据 `startTime`/`endTime` 自行显示倒计时、即将开始、进行中、已结束等状态
6. **下单后流程**：秒杀下单成功后返回的是普通订单，后续支付、查看订单等操作与普通订单完全一致
