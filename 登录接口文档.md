## 登录注册接口文档（网关版）

**基础信息**
- 网关地址：`http://localhost:10087`
- 所有请求统一走网关，由网关自动路由到各微服务
- 认证方式：JWT Token，请求头 `Authorization: Bearer <token>`
- Token 有效期：7 天

---

### 一、后台管理员登录

#### 1.1 管理员登录

`POST /mall-sso/admin/sso/login`

**请求体** (`application/json`)

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|---|---|---|---|---|
| username | string | 是 | 4~16位字母数字，首字符必须为字母 | 用户名 |
| password | string | 是 | 4~16位可打印ASCII字符 | 密码 |

**请求示例**

```json
{
  "username": "root",
  "password": "123456"
}
```

**成功响应** `state: 200`

```json
{
  "state": 200,
  "message": null,
  "data": {
    "tokenHeader": "Bearer",
    "tokenValue": "eyJhbGciOiJIUzUxMiJ9.eyJzdWIiOi..."
  }
}
```

**失败响应**

```json
// 校验失败 state: 400
{
  "state": 400,
  "message": "登录失败！用户名不允许为空！",
  "data": null
}

// 认证失败 state: 401
{
  "state": 401,
  "message": "登录失败！用户名或密码错误！",
  "data": null
}
```

---

#### 1.2 管理员登出

`POST /mall-sso/admin/sso/logout`

**请求头**

| 字段 | 值 |
|---|---|
| Authorization | Bearer `<token>` |

**成功响应**

```json
{
  "state": 200,
  "message": null,
  "data": null
}
```

### 三、用户注册

#### 3.1 检查重复值

`POST /mall-ums/ums/user/checkValue`

> 建议在注册表单中实时调用，用于校验用户名/手机号/邮箱是否已被占用。

**请求参数** (`application/x-www-form-urlencoded`)

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| value | string | 是 | 待校验的值 |
| type | string | 是 | 校验类型：`phone` / `username` / `email` |

**请求示例**

```
POST /mall-ums/ums/user/checkValue
Content-Type: application/x-www-form-urlencoded

value=jackson&type=username
```

**成功响应**（未重复）

```json
{
  "state": 200,
  "message": null,
  "data": null
}
```

**失败响应**（已重复，state: 409）

```json
{
  "state": 409,
  "message": "该用户名已被注册！",
  "data": null
}
```

---

#### 3.2 用户注册

`POST /mall-ums/ums/user/register`

**请求体** (`application/json`)

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|---|---|---|---|---|
| username | string | 是 | 4~16位字母数字，首字符为字母 | 登录用户名 |
| nickname | string | 是 | 2~16个字符 | 昵称 |
| password | string | 是 | 4~16位可打印ASCII字符 | 密码 |
| ackPassword | string | 是 | 4~16位可打印ASCII字符 | 确认密码（需与 password 一致） |
| email | string | 是 | 标准邮箱格式 | 邮箱 |
| phone | string | 是 | 11位手机号，1开头 | 手机号 |

**请求示例**

```json
{
  "username": "jackson",
  "nickname": "杰克逊",
  "password": "123456",
  "ackPassword": "123456",
  "email": "jackson@example.com",
  "phone": "13800138000"
}
```

**成功响应**

```json
{
  "state": 200,
  "message": null,
  "data": null
}
```

**失败响应**

```json
// 校验失败 state: 400
{
  "state": 400,
  "message": "注册用户失败，请填写正确的手机号！",
  "data": null
}

// 数据冲突 state: 409
{
  "state": 409,
  "message": "该用户名已被注册！",
  "data": null
}
```

### 四、前端校验规则汇总

前端可同步做表单校验，避免无效请求：

| 字段 | 正则表达式 | 说明 |
|---|---|---|
| 用户名 | `^[a-zA-Z][0-9a-zA-Z]{3,15}$` | 4~16位，首字母 |
| 密码 | `^[\u0020-\u007e]{4,16}$` | 4~16位，可打印ASCII |
| 昵称 | `^.{2,16}$` | 2~16个字符 |
| 邮箱 | `^[a-zA-Z0-9_-]+@[a-zA-Z0-9_-]+(\.[a-zA-Z0-9_-]+)+$` | 标准邮箱 |
| 手机号 | `^1[34589]\d{9}$` | 11位手机号 |

---

### 五、前端对接流程

```
1. 注册页面
   ├── 输入用户名 → 调用 checkValue(username) 实时校验
   ├── 输入手机号 → 调用 checkValue(phone) 实时校验
   ├── 输入邮箱   → 调用 checkValue(email) 实时校验
   └── 提交表单   → 调用 register()

2. 登录页面
   └── 提交表单   → 调用 login() → 存储 Token → 跳转首页

3. 登录后
   ├── 每次进入页面 → 调用 getUserInfo() 获取用户信息
   └── 退出登录     → 调用 logout() → 清除 Token → 跳转登录页
```

### 六、统一响应结构

所有接口返回相同的 `JsonResult` 结构：

```typescript
interface JsonResult<T = any> {
  state: number     // 200=成功, 400=校验失败, 401=未认证, 409=冲突
  message: string   // 失败时的错误描述，成功时为 null
  data: T           // 业务数据，失败时为 null
}
```

前端 Axios 拦截器根据 `state` 字段判断：`200` 取 `data`，非 `200` 取 `message` 显示错误。
