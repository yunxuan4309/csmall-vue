# 用户注册功能 — 前端接口文档

> 对应后端服务：mall-ums（用户管理服务）
> 更新时间：2026-05-19

---

## 一、接口清单

| 接口 | 方法 | 说明 | 是否需要登录 |
|------|------|------|------------|
| `/ums/user/register` | POST | 用户注册 | 否 |
| `/ums/user/checkValue` | POST | 校验用户名/手机/邮箱是否已存在 | 否 |

---

## 二、接口详情

### 2.1 用户注册

**请求**

```http
POST /ums/user/register
Content-Type: application/json

{
  "username": "testuser123",
  "nickname": "测试用户",
  "email": "testuser@example.com",
  "phone": "13800138000",
  "password": "mypassword123",
  "ackPassword": "mypassword123"
}
```

**请求参数字段说明**

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|----------|------|
| `username` | string | 是 | 4~16位，字母开头，只能包含字母和数字 | 用户名 |
| `nickname` | string | 是 | 2~16位任意字符 | 昵称 |
| `email` | string | 是 | 标准邮箱格式 | 电子邮箱 |
| `phone` | string | 是 | 11位手机号，1[34589]开头 | 手机号码 |
| `password` | string | 是 | 4~16位可打印字符 | 密码 |
| `ackPassword` | string | 是 | 与 password 值一致 | 确认密码 |

**成功响应（200）**

```json
{
  "state": 200,
  "message": "OK",
  "data": {
    "id": 123456789,
    "username": "testuser123",
    "nickname": "测试用户"
  }
}
```

**data 字段说明**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | long | 新用户的 ID（雪花算法生成），前端可用于登录后本地缓存 |
| `username` | string | 用户名 |
| `nickname` | string | 昵称 |

**错误响应**

```json
// 参数校验失败（400）
{
  "state": 400,
  "message": "注册用户失败，请填写正确的手机号！"
}

// 用户名已存在（409）
{
  "state": 409,
  "message": "注册用户名已存在"
}

// 手机号已存在（409）
{
  "state": 409,
  "message": "注册手机号已存在"
}

// 邮箱已存在（409）
{
  "state": 409,
  "message": "注册邮箱已存在"
}

// 密码与确认密码不匹配（400）
{
  "state": 400,
  "message": "密码和确认密码不相等"
}
```

---

### 2.2 校验值是否重复

用于注册页面的「实时校验」，当用户输入完用户名/手机号/邮箱后，可立即调用此接口判断是否已被使用。

**请求**

```http
POST /ums/user/checkValue?value=testuser123&type=username
Content-Type: application/x-www-form-urlencoded
```

**请求参数**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `value` | string | 是 | 要校验的值 |
| `type` | string | 是 | 校验类型：`username` / `phone` / `email` |

**成功响应（200）— 值可用**

```json
{
  "state": 200,
  "message": "OK",
  "data": null
}
```

**错误响应（409）— 值已存在**

```json
{
  "state": 409,
  "message": "注册用户名已存在"
}
```

## 三、开发环境接入

### 后端地址

| 环境 | 说明 |
|------|------|
| 开发环境 | 直接调 UMS 服务：`http://localhost:10006/ums/user/register` |
| Gateway 方式 | `http://localhost:10087/ums/user/register`（dev/ test 环境均可） |

### 前端请求示例（axios）

```javascript
// 1. 校验用户名是否已存在
axios.post('/ums/user/checkValue', null, {
  params: { value: 'testuser', type: 'username' }
}).then(res => {
  // state: 200 → 可用；state: 409 → 已存在
});

// 2. 提交注册
axios.post('/ums/user/register', {
  username: 'testuser123',
  nickname: '测试用户',
  email: 'test@example.com',
  phone: '13800138000',
  password: 'mypassword123',
  ackPassword: 'mypassword123'
}).then(res => {
  // res.data.state: 200 → 注册成功
  // res.data.data.id → 新用户ID
});
```

## 四、生产环境说明

生产环境通过 API 网关统一入口 `http://8.156.85.160` 访问，Nginx → Gateway 路由规则：

| 路径 | 目标服务 |
|------|----------|
| `/ums/user/register` | mall-ums |
| `/ums/user/checkValue` | mall-ums |

**生产环境请求示例：**

```bash
curl -X POST http://8.156.85.160/ums/user/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser123",
    "nickname": "测试用户",
    "email": "test@example.com",
    "phone": "13800138000",
    "password": "mypassword123",
    "ackPassword": "mypassword123"
  }'
```
