## 一、秒杀功能相关问题

### 1.1 秒杀商品列表不显示商品

**问题描述**: 后端返回数据正确（分页显示6条），但前端列表为空

**原因分析**: 
- 不同微服务返回的分页数据字段名不一致
- 商品/订单服务使用 `res.data.list`
- 秒杀服务（MyBatis-Plus IPage）文档写的是 `res.data.records`
- 实际后端统一使用了 `list` 字段

**解决方案** (`SeckillList.vue:147`):
```javascript
// 兼容两种格式
productList.value = res.data.list || res.data.records || []
```

---

### 1.2 秒杀订单提交失败 - 座机号码格式错误

**问题描述**: 提交秒杀订单时后端返回 "请输入正确座机号码"

**原因分析**: 
- `telephone` 字段在前端表单中是选填项
- 用户未填写时传了空字符串 `""`
- 后端对空字符串进行了座机号格式校验，导致失败
- 同理 `streetCode` 也可能遇到此问题

**解决方案** (`SeckillDetail.vue:804,811`):
```javascript
const orderData = {
  // ...
  telephone: orderForm.value.telephone || null,  // 空字符串转为null
  streetCode: orderForm.value.streetCode || null,
  // ...
}
```

### 1.3 秒杀列表点击卡片跳转不一致

**问题描述**: 
- 点击"查看详情"按钮 → 进入秒杀详情页（正确）
- 点击商品图片/标题 → 进入普通商品详情页（错误）

**原因分析**: 图片和标题绑定的点击事件是 `goToDetail`（跳 `/product/${id}`），而按钮是 `goToSeckillDetail`（跳 `/seckill/${id}`）

**解决方案** (`SeckillList.vue:38,54`):
```javascript
// 统一改为跳转到秒杀详情页
@click="goToSeckillDetail(product.id)"
```

---

### 1.4 秒杀订单数据结构不匹配

**问题描述**: 秒杀下单接口返回错误，订单数据结构不符合API要求

**原因分析**:
- `submitSeckillOrder` 使用了 `Content-Type: application/x-www-form-urlencoded`
- 但 API 要求 JSON 格式，且需要嵌套对象 `seckillOrderItemAddDTO`
- 缺少 `spuId` 字段

**解决方案**:
1. `seckill.js:47-49`: 移除 form-urlencoded，改用默认 JSON
2. `SeckillDetail.vue:800-830`: 重构订单数据结构
```javascript
const orderData = {
  spuId: product.value.id,  // 补充缺失字段
  // ...地址信息
  seckillOrderItemAddDTO: {  // 嵌套对象格式
    skuId: selectedSku.value.id,
    title: selectedSku.value.title,
    // ...
  }
}
```

### 1.5 秒杀特有错误未处理

**问题描述**: 秒杀失败时提示信息不友好

**解决方案** (`SeckillDetail.vue:835-850`): 增加特定状态码处理
```javascript
catch (error) {
  const msg = error.response?.data?.message || ''
  if (error.response?.status === 403 || msg.includes('已经购买过')) {
    ElMessage.error('您已经购买过该商品，秒杀商品限购1件')
  } else if (error.response?.status === 400 || msg.includes('售罄')) {
    ElMessage.error('抱歉，该商品已售罄')
  } else if (error.response?.status === 404 || msg.includes('随机码')) {
    ElMessage.error('秒杀链接已失效，请刷新页面重试')
  } else if (error.response?.status === 500 || msg.includes('服务器忙')) {
    ElMessage.warning('当前访问人数过多，请稍后再试')
  }
}
```

## 三、Element Plus 组件警告

### 3.1 ElPagination total 类型错误

**问题描述**: `Invalid prop: type check failed for prop "total". Expected Number, got String`

**原因**: 后端返回的 `total` 是字符串 "7"，组件要求 Number

**解决方案**: 多处文件统一转换
```javascript
total.value = Number(res.data.total) || 0
```

**涉及文件**:
- `ProductList.vue`
- `OrderList.vue`
- `SeckillList.vue`

---

### 3.2 ElRadio label 弃用警告

**问题描述**: `[ElRadio] [API] label has been deprecated, please use value instead`

**解决方案**: 将所有 `:label="1"` 改为 `:value="1"`，`label="家"` 改为 `value="家"`

**涉及文件**:
- `OrderSettle.vue` (支付方式、地址标签)
- `OrderPay.vue` (支付方式)
- `SeckillDetail.vue` (支付方式)
